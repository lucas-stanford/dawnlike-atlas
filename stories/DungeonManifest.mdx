# Dungeon Manifest

The `DungeonManifest` is the single configuration object passed to
[`generateDungeon(manifest)`](https://github.com/lucas-stanford/dawnlike-atlas/blob/master/src/phaser/generators/dungeon.js)
to produce one deterministic dungeon level. Spread
[`DEFAULT_DUNGEON_MANIFEST`](https://github.com/lucas-stanford/dawnlike-atlas/blob/master/src/phaser/generators/dungeon.js)
into your own object to inherit defaults for the fields you don't set.

```js
import { generateDungeon, DEFAULT_DUNGEON_MANIFEST }
  from 'dawnlike-atlas/roguelike/generators';

const level2 = generateDungeon({
  ...DEFAULT_DUNGEON_MANIFEST,
  seed: 12345,
  level: 2,
  width: 60,
  height: 40,
  roomWidth: [4, 9],
});
```

The live Dungeon example exposes the same knobs as Storybook controls — open
[Zone Examples / Dungeon](?path=/story/dawnlike-zone-examples-dungeon--dungeon)
to play with them visually.

## Returns

```ts
{
  width:    number,
  height:   number,
  tiles:    Array<Array<Tile>>,                     // tiles[y][x]
  markers:  { stairsUp:{x,y}, stairsDown:{x,y}|null },
  walkable: (x:number, y:number) => boolean,
  manifest: DungeonManifest,
}
```

Each tile is `{ wall: boolean, floor: boolean, marker: 'stairsUp' | 'stairsDown' | null }`.
Levels at or beyond `bottomLevel` have `stairsDown: null` — that's the bottom
of the dungeon.

Any field left `undefined` is omitted from the options object passed to
`ROT.Map.Digger`, so rot-js applies its own default — useful when you only
want to override one of the digger knobs.
