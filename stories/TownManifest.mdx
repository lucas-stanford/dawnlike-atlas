# Town Manifest

The `TownManifest` is the single configuration object passed to
[`generateTown(manifest)`](https://github.com/lucas-stanford/dawnlike-atlas/blob/master/src/phaser/generators/town.js)
to produce one deterministic town. Spread
[`DEFAULT_TOWN_MANIFEST`](https://github.com/lucas-stanford/dawnlike-atlas/blob/master/src/phaser/generators/town.js)
into your own object to inherit defaults for the fields you don't set.

```js
import { generateTown, DEFAULT_TOWN_MANIFEST }
  from 'dawnlike-atlas/roguelike/generators';

const town = generateTown({
  ...DEFAULT_TOWN_MANIFEST,
  seed: 42,
  buildingCount: { min: 8, max: 10 },
  flowers: { density: 0.12 },
});
```

The live Town example exposes the most-used knobs as Storybook controls — open
[Zone Examples / Town](?path=/story/dawnlike-zone-examples-town--town)
to play with them visually.

## Returns

```ts
{
  width:    number,
  height:   number,
  tiles:    Array<Array<Tile>>,         // tiles[y][x]
  markers:  { worldExit:{x,y} },
  walkable: (x:number, y:number) => boolean,
  manifest: TownManifest,
}
```

Each tile has fields
`{ type, street, wall, floor, door, doorSide, tree, decor, fountain, marker, sign, furniture, npc, flower, buildingType }`
— see
[`renderTownTile`](https://github.com/lucas-stanford/dawnlike-atlas/blob/master/src/phaser/autotileRender.js)
for the rendering contract.

### Default `buildingTypeWeights`

```js
{ home: 3, inn: 1, smithy: 1, shop: 1, church: 1 }
```

## Nested configs

### `TownNpcConfig`

  <thead>

      <th align="left">Description</th>
    </tr>
  </thead>
  <tbody>
    <tr><td><code>chance</code></td><td><code>number</code></td><td><code>0.8</code></td><td>Per-building probability of placing any NPCs.</td></tr>
    <tr><td><code>perBuilding</code></td><td><code>{'{min,max}'}</code></td><td><code>{'{1,3}'}</code></td><td>NPC count range per building (capped at free floor tiles).</td></tr>
    <tr><td><code>palettes</code></td><td><code>object</code></td><td>see below</td><td>Per-buildingType NPC sprite name palette. <code>default</code> is the catch-all.</td></tr>
  </tbody>
</table>

Default palettes:

```js
{
  home:    ['peasant man', 'peasant woman'],
  inn:     ['peasant man', 'peasant woman', 'farmer man', 'farmer woman'],
  smithy:  ['miner', 'peasant man'],
  shop:    ['gnome wizard', 'peasant woman', 'peasant man'],
  church:  ['priest', 'monk', 'aligned priest'],
  default: ['peasant man', 'peasant woman'],
}
```
